Animation Basics

Understanding the fundamentals of CSS animations

What are CSS Animations?

CSS animations allow you to animate transitions between CSS property values over time. They consist of two main components:

  • @keyframes: Define the animation sequence
  • Animation properties: Control timing, duration, and behavior

Unlike transitions, animations can run automatically and loop indefinitely, making them perfect for complex motion graphics.

Transitions vs Animations

Transitions

Animate between two states when triggered by user interaction (hover, focus, etc.)

Animations

Can run automatically, loop, and define multiple intermediate states using keyframes

Best Practice: Use transitions for simple state changes and animations for complex sequences.

Animation Properties

Master all the properties that control animation behavior

1

animation-name

Specifies the name of the @keyframes rule to use.

animation-name: slideIn;
2

animation-duration

Sets how long the animation takes to complete one cycle.

animation-duration: 2s;
3

animation-timing-function

Controls the speed curve of the animation.

animation-timing-function: ease-in-out;
4

animation-delay

Specifies when the animation should start.

animation-delay: 0.5s;
5

animation-iteration-count

Sets how many times the animation should repeat.

animation-iteration-count: infinite;
6

animation-direction

Controls whether the animation plays forward, backward, or alternates.

animation-direction: alternate;

Timing Functions Explained

ease

Slow start, fast middle, slow end

ease-in

Slow start, accelerates

ease-out

Fast start, decelerates

linear

Constant speed

Keyframes Deep Dive

Create complex animation sequences with @keyframes

Basic Keyframe Syntax

@keyframes animationName {
  0% {
    transform: translateX(0);
    opacity: 0;
  }
  50% {
    transform: translateX(50px);
    opacity: 1;
  }
  100% {
    transform: translateX(100px);
    opacity: 0;
  }
}

Tip: You can use percentages (0%, 50%, 100%) or keywords (from, to) to define keyframe points.

Multiple Properties

@keyframes complexMove {
  0% {
    transform: translateX(0) rotate(0deg);
    background-color: #3b82f6;
    border-radius: 0%;
  }
  50% {
    transform: translateX(100px) rotate(180deg);
    background-color: #ef4444;
    border-radius: 50%;
  }
  100% {
    transform: translateX(0) rotate(360deg);
    background-color: #10b981;
    border-radius: 0%;
  }
}

Pro Tip: Animate multiple properties simultaneously for rich, complex effects.

Animation Chaining

Create sequences by using animation delays to chain multiple animations together.

.element1 { animation: fadeIn 1s ease-out; }
.element2 { animation: fadeIn 1s ease-out 0.5s; }
.element3 { animation: fadeIn 1s ease-out 1s; }

CSS Variables in Animations

Use CSS custom properties to create dynamic, reusable animations.

:root {
  --animation-duration: 2s;
  --animation-delay: 0.5s;
}

.animated {
  animation: slideIn var(--animation-duration) 
             ease-out var(--animation-delay);
}

Benefit: Easily modify animation timing across your entire application by changing CSS variables.

Animation Fill Modes

forwards

Element retains final keyframe styles after animation ends

backwards

Element applies first keyframe styles during delay period

both

Combines forwards and backwards behavior

Common Animation Patterns

Ready-to-use animation recipes for common UI interactions

Loading Spinner

@keyframes spin {
  to { transform: rotate(360deg); }
}
.spinner {
  animation: spin 1s linear infinite;
}

Fade In Up

Text
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(30px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }

Heartbeat

@keyframes heartbeat {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.1); }
}
.heartbeat {
  animation: heartbeat 1.5s ease-in-out infinite;
}
